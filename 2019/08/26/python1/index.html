파이썬 개요


1991년에 발표된 인터프리터 방식[3]의 프로그래밍 언어.

  • 공식 홈페이지
  • 공식 깃헙
  • 나무 위키 소개

1. 탄생 배경

창시자는 귀도 반 로섬(Guido van Rossum, 네덜란드어 발음으로 ‘히도 판 로쉼’)
1989년 크리스마스 주에, 연구실이 닫혀있어서 심심한 김에 만들었다고 한다.
이름은 귀도가 즐겨 보던 영국의 6인조 코미디 그룹 “몬티 파이선”에서 따왔다. 사전적 의미를 따라 심볼마크가 뱀 모양이다.

python은 인터프리터 언어이다. 즉 일반적인 컴파일 언어와 차이가 있다.

그렇다면 우선적으로 인터프리터란 무엇인지 알아보자:

인터프리터란?

1. 인터프리터와 컴파일러의 차이:

  • 컴파일언어와 인터프린터 언어의 가장 큰 차이점은 컴파일 시점 이다.
    (런타임 전에 컴파일을 하는지 여부에 따라 판단됨)

컴파일러는 전체소스코드를 보고 명령어를 수집 및 재구성하는 하며, 일반적으로 고레벨언어를 바로 기계어로 변환한다.

인터프리터는 소스코드의 각 행을 연속적으로 분석하며 실행하고,
고레벨 언어를 바로 기계어로 변환하는 것이 아닌 중간형태로 변환 한 다음 실행한다.

2. 인터프리터의 특징

  1. 인터프리터는 고레벨 언어를 중간 코드(intermediate code)로 변환하고 이를 각 행마다 실행한다. 이 중간 코드는 다른 프로그램에 의해 실행된다.
  2. 일반적으로 각 행마다 실행하는 특성을 가진 인터프리터 언어의 실행시간은 컴파일 언어에 비해 다소 느리다.
  3. 컴파일러는 전체 소스코드를 변환 한 뒤 에러를 보고하는 반면, 인터프리터는 각 행 마다 실행하는 중, 에러가 보고되면 이후 작성된 코드를 확인 하지 않는다.

3. 언어별 속성

  • 컴파일 언어: C, C++
  • 인터프리터 언어: python
  • 컴파일러+인터프리터 혼용: java
    (* java는 런타임 이전 컴파일을 통해 기계어가 아닌 바이트어로 변경되지만 컴파일 시점 때문에 컴파일 언어라고 함)
