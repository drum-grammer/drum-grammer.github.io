딥러닝 level1 - 7 딥러닝은 어떻게 대세가 될 수 있었나?
Created 2019-11-02| Updated 2019-11-06|4. MACHINE LEARNINGMachine Learning Foundation
Post View:

딥러닝의 기초의 인공신경망 알고리즘은 1950년부터 존재했는데, 왜 2000년대에 들어서 폭팔적인 인기를 끌고 획기적인 발전을 이루게 되었을까? 소위 기술 흐름에 대세가 되었을까?

여러가지 이유가 있겠지만, 가장 주요한 원인을 알아보자:

1. 데이터

2000년 이전까지는 연구자가 인공신경망에 활용할 수 있는 데이터를 구하는 것은 매우 어려운 일이었다.

음성인식 분야에서 획기적인 발전은 준 switchboard의 데이터 크기는 기가바이트 수준이었고, IMAGENET 나오기 전까지는 이미지 인식에 활용 될 수 있는 데이터는 전무한 상황이었다.

하지만 모바일 디바이스의 대중화에 따라, 사람들은 모바일 디바이스를 통해, 소셜 플랫폼에 자발적으로 여러 종류의 데이터를 제공하게 되었다. 심지어 웨어러블 디바이스 까지 보편화 되면서, 인간은 노력없이, 여러 종류의 정보들을 데이터화 할 수 있게 되었다.

요즘에는 카카오나 네이버 라인에서 사용자에 의해 쌓이는 채팅 일일 데이터양은 테라바이트 수준이다. 1990년도부터 2017년까지의 모든 한국 뉴스 정보는 수천만건 정도이고 용량은 60기가정도 인 것을 감안하면, 과거에 비해 얼마나 폭팔적인 데이터들이 쌓이고 있는지 알 수 있다.

더 이상 데이터양이 부족하지 않는 시대가 열리자, 최소한 수천만 이상의 데이터에서 자신의 성능을 발휘하게 되는 인공신경망 알고리즘은 비로소 물 만난 물고기 처럼, 자신의 장점을 마음껏 뽐낼수 있게 된 것이다.

2 하드웨어의 발전 - GPU

방대한 데이터량이 확보되자, 연구자들은이 데이터를 주어진 시간 안에 정확하게 학습시키는 데 어려움을 느끼게 되었다.
2010년 이전에는, 데이터가 일정 사이즈가 커지면 vanish gradient 등 여러 현상들이 발생하면서 결국 제대로 학습 하지 못하는 문제가 있었다.

학습 시간은 오래 걸리고 성능도 제대로 나오지 않는 기술 장벽에 부딪치게 된다.

이를 보완하기 위해 여러 알고리즘 연구가 진행되었다.
볼츠만머신이나 오토 인코더처럼 데이터 정보를 인코딩/압축하여 딥러닝 알고리즘에 넣거나, 딥러닝 알고리즘 일부분을 미리 학습시켜서 전체를 묶은 후 다시 한번 학습시키는 pre-training 와 같은 방법으로 문제를 개선 시키수 있는 여러 방법론들이 발명되었다.
관련 링크1 - 딥러닝
관련 링크2 - 볼쯔만 머신
관련 링크3 - 오토인코더

이러한 어려움에 대한 하드웨어적 해결책이 되어준 것이 바로 GPU 이다.
GPU(그래픽 처리 장치, Graphics Processing Unit)는 메모리를 빠르게 처리하고 바꾸어 화면으로 출력할 프레임 버퍼 안의 영상 생성을 가속하도록 설계된, 전문화된 전자 회로로, 고성능 게임을 처리하기 위해 나온 프로세서이다.

딥러닝 연구 분야에 이 GPU가 사용되면서, 딥러닝은 비약적인 발전을 하게 된다.
고성능 게임의 대중화, 딥러닝 및 비트코인 산업에서의 폭넓은 응용으로 인해 GPU 또한 함께 비약적인 발전을 하게 되었다.

2010년도부터, GPU가 딥러닝에 응용 되기 시작하였고,
더욱 강력한 하드웨어 서포트를 받기위해 더욱 강력한 GPU를 개발하여 활용하였습니다.

2016년 5월, 구글에서 벡터/행렬연산의 병렬처리에 특화되 있는 TPU(Tensor Processing Unit)를 발표하고, 이를 데이터 분석 및 딥러닝에 활용하기 시작했다.

결론

방대한 데이터와 GPU의 하드웨어 서포트, 그리고 AI Winter를 이겨낸 여러 인공지능 공학자들의 공헌으로, 딥러닝은 기술의 중심에 서게 되었다.
