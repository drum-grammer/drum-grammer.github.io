협업 시 필요한 공용계정 비밀번호 관리 툴 "PASS"
Created 2019-11-20| Updated 2019-11-20|5. OPERATE SYSTEMLinux
Post View:

1. 필요성

  • 개발 프로젝트 진행 시, 공용 계정 사용하게 됨.
  • 많은 구성원과, 다양한 공용 계정의 Password를 안전하고 정확하게 저장해야 함.
  • 또한 이를 공용 계정의 Password를 안전하지만 간편하게 공유할 수 있어야 함.
  • 위 기능을 가진 무료 툴, “pass” 도입

2. 사용 툴 (pass) 개요

pass 동영상 강의 - 영문

  • 툴 정의: the standard unix password manager
  • 비밀 번호는~/.password-store 경로에 저장됨.
  • 사용자 친화적인 command 구성
    PASS Command Manual

3. 설정 flow

차조 링크

1) gpg key 생성

gpg는 비대칭키 방식의 암호화를 사용할 수 있는 오픈 소스 툴이다.
(이미 생성한 gpg key가 있다면, 기존 키를 그대로 사용 가능)

gpg를 이용해서 키를 생성하면
공개 키(public key)와 비밀 키(prviate key)가 함께 생성되고,
~/.gnupg/ 디렉토리에 저장됨

  • 자신의key pair의 ID (gpg-id) 확인 하는 법
