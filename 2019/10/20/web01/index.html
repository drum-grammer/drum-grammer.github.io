What is CLOUD MESSAGING?

다양한 디바이스에 원하는 메시지 발송을 도와 주는 서비스 이다.
클라우드 메시징을 통하여 원하는 데이터/알림 메시지를 다양한 기기에 발송할 수 있다.

주요하게 FCM(Firebase Cloud Messaging)과 GCM(Google Cloud Messaging)이 있다.

GCM(google cloud messaging) 과 FCM(firebase cloud messaging)

GCM (google cloud messaging)

구글에서 제공하는 서버에서 애플리케이션으로 푸시 메시지를 보낼 수 있는 서비스이다.
Android, IOS를 지원한다.

  • 전송 주소 : https://android.googleapis.com/gcm/send

FCM (firebase cloud messaging)

GCM의 새버전이다.
Android & IOS & Mobild Web을 지원한다.

  • 전송 주소 : https://fcm.googleapis.com/fcm/send

서버에서 클라이언트 애플리케이션으로 메시지(알림)를 안정적으로 보낼 수 있는 교차 플랫폼 메시징 솔루션이다.
모든 사용자에게 알림을 전송할수도있고, 그룹을 지어 알림을 전송할 수 있다.
Firebase 클라우드 메시징을 통해 앱 서버에서 클라이언트 앱으로 메시지를 전달한다.

구현 프로세스

GCM 구현 방법

  • GCM 서버와의 통신에 필요한 추가 패키지 설치
  • GCM 서버에 대한 액세스 권한 구성
  • Google Play 서비스의 존재 여부를 확인하는 코드 구현
  • GCM 등록 토큰에 대한 협상하는 등록의도 서비스 구현
  • GCM 등톡 토큰 업데이트를 수신하는 인스턴스 ID 수신기 서비스 구현
  • GCM 통해 앱 서버에서 원격 메시지 수신하는 GCM 수신기 서비스 구현

FCM 개요

전송, 라우팅 및 서버 응용 프로그램 및 모바일 클라이언트 앱 사이 메시지 큐를 처리하는 플랫폼.

  • FCM 등록 (사용자는 FCM을 사용하기 전에 등록해야함.)
  • 클라이언트 앱 FCM에 보낸사람 ID, API키, 앱 ID 전달하는 등록 토큰을 가져오기 위해 FCM과 연결
  • FCM등록 토큰을 리턴 (클라이언트 앱으로)
  • 클라이언트 앱은 서버에 등록 토큰을 전달
