What is ETL?

  • Extract Transfer Load
  • Extract: 데이터를 외부 소스에서 긁어 오고
  • Transfer: 우리 시스템에 맞게 가공 하고
  • Load: 우리 데이터 베이스에 저장

What is Data Pipeline

  • ETL과 비슷한 느낌.
  • 요즘은 더 간지나게 data pipeline 이라고 함.
  • 파이프 가 뭐가 들어와서 지나가고 나가는것 처럼 (배수 파이프 처럼) 데이터가 외부에서 들어와서 우리 내부 시스템으로 들어오는걸 파이프에 빗대에서 이야기 하는 것임.
  • 즉 ETL과 비슷.
  • 간단한 data pipeline은 파이썬 스크립트 파일로 구현해도 됨.
  • 하지만 요즘 왠만한 시스템은 여러 외부 소스에서 데이터를 구현하고 또 데이터 가 서로 dependency가 걸리게 때문에 굉장히 복잡해짐. 그래서 단순 스크립트 로는 구현 하기 까다로움.
  • 그렇게 때문에 AirFlow나 luigi같은 프레임워크를 사용하는 것임.

Data Pipeline 중요 3가지 요소

  1. Job
  2. Scheduling
  3. Dependency

What is AirFlow

  • AirBnb에서 만든 worflow management 프레임워크.
  • workflow 는 뭐든 될 수 있기 때문에 (예를 들어 단순한 sleep 5 문), 꼭 데이퍼 파이프라인 전용이라고 할 수 는 없지만 일반적으로 데이터 파이프라인 시스템 구현에 많이 쓰이고, AirBnb 도 그것때문에 만든것임.

How to set up and use AirFlow

1. Install

  • pip install apache-airflow

2. 설정

  • AIRFLOW_HOME 환경 변수를 설정. 만일 설정 하지 않으면 디포트로 홈 디렉토리에 airlfow 라는 디렉토리로 설정 됨.
  • AIRFLOW_HOME 디렉토리로 cd 한후 에 airflow initdb 실행해서 airflow db를 초기화.
    • 참고로, 여기서 초기화 하는 db는 airflow 관련 데이터(dag 실행 내역등) 을 저장하는 데이터이고 실제 시스템 데이터랑은 관련이 없다.
    • 디포트로 sqlite3 db가 사용됨. 추후에 mysql 혹은 다른 더 안정적인 db로 변경 필요함.

3. Dag 파일 생성.

  • 생성한 dag 파일은 AIRFLOW_HOME/dags 디렉토리에 저장.
  • python <dag file name>.py 명령어로 만든 dag 파일에 파이썬적인 에러가 없는지 확인.
  • 그 후, airflow scheduler 명령어 실행.
  • 그 다음 airflow list_dags 명령어 실행하여 해당 dag가 리스트 되어 나오는지 확인.
  • 마지막으로, airflow webserver 명령어를 실행하여 airflow UI 화면 실행 후 해당 dag를 켜준다.
  • 이제 해당 dag가 오류 없이 돌아가나 확인.
